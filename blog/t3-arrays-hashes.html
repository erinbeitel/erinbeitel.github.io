Array vs Hash

What are they and when do you use them?

June 7, 2015

In the first week of DBC, we focused on the command line; in the second week, HTML and CSS were introduced; this week we started Ruby. Amongst the excitement: arrays and hashes.

What are they are when do you use them?

Great question... Lets start with their similarities. Arrays and Hashes both store stuff, object type elements to be specific. They also both allow elements to be retrieved using a "key". And, they both allow methods to be called on them.

How you retrieve what you've stored differs between arrays and hashes. When retrieving an element of an array, the key you use is an integer because arrays always have consequitive integer index keys. For example, names[0] would get you the first element of the array "names". If that array was names = ["sara", "erykah", "sam"] then, names[0] would return "sara".

To retrieve something stored in a hash, the key is a object. For example, in the hash "address", if you used the key "street_address" to store "park" then to access your street address you would type address['street_address'] which would return "park".

How do you choose?

Arrays have the advantage of having easily accessible elements. If you wanted to iterate through the different elements of an array and access each index using an integer. You wouldn't be able to do this the same way on hashes that have non-integer keys. Hashes, although slightly more challenging to access, are useful because of the flexibilty that they provide for storing information. The key-value pair for a hash can have an integer key (ex: '5'), a string key (ex: 'hats'), a floating point key (ex: 5.55) or any other Ruby object as its key. It's common to think of a hash like a dictionary, where the key is the word that allows you to look up the associated definition.

Declaring arrays and hashes looks quite different. To declare an array, literally, you could do this:

array = ["a", "b", "c"]

array = []

or, to declare it using the array object's constructor, array.new, you could do this:

array = Array.new

A hash can be declared literally, like this:

address = {street_address: "park", city: "chicago"}

address = {:street_address=>"park", :city=>"chicago"}

or declaratively, using a new method, like this:

address = Hash.new

address[:street_address] = "park"

address[:city] = "chicago"

In some cases, both arrays and hashes are applicable but one is more advantagous. For example, if you were storing the owners of cars using their 8 digit liscense plate number, or the names of students using their 8 digit ID numbers, you would use a hash. If you used an array for this scenario, you would end up with a 100,000,000 element table to access the names associated with the 8 digit numbers. Not ideal.

For more details, see the Ruby documentation on Hashes and Arrays and this blog about How the Hash Works.
