Ruby Hashes vs. JavaScript Objects

Concepts Compared

July 13, 2015

In Ruby, a hash looks like this:

example_hash = { first_key: first_value , second_key: second_value}

In JavaScript, an object looks like this:

var objectName = { first_name: first_value , first_name: first_value};

Except for the variable definition, "var", these two definitions are extremely similar. The semicolon is optional for this isolated object in JavaScript but it should be added to help it play nice with others in the context of other code.

In Ruby, the elements of a hash can be retrieved as so, for example:

example_hash[:first_key] —-> first_value

In JavaScript, the elements of an object can be retrieved like this, for example:

objectName.first_name —-> first_value

For more about the similarities and differences between these two languages, checkout this Medium article: Ruby and Javascript Comparison.
